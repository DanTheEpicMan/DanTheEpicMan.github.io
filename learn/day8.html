Day 8 - Namespaces

What if we wanted to have a lot of variables that basically have the same name

For instance, if we had two variables, one for an employee Laptop ID and one for an Admin Laptop ID, maybe even more variables could overlap, username, password and a bunch of other stuff.

The best way to do this is to seperate the variables into what are called namspaces

namespace Employee { int laptopID = 456; std::string username; std::string password; } namespace Admin { int laptopID = 123; std::string username; std::string password; } ...in main... Admin::laptopID = 54321; std::cout << Employee::laptopID << std::endl; Output: 456

As you can see, they work just as normal variables, execpt you need to do the name of the namespace followed by ::

Namespaces inside a Namespace

namespace Employee { int laptopID; std::string username; std::string password; namespace MicrosoftWord { std::string fileName; } } ...in main... Employee::MicrosoftWord::fileName = "document.docx"; std::cout << Employee::MicrosoftWord::fileName << std::endl; Output: document.docx

Cheatsheet Day 8

        Example:
        namespace Employee {
            int laptopID;
            string username;
            string password;
        }

        Accessing the variables:
        Employee::laptopID = 12345;
        Employee::username = "employee";
        Employee::password = "password123";

        Vocab:
        Namespace: A declarative region that provides a scope to the identifiers (the names of types, functions, variables) inside it.
        Scope: The region of the program where a variable or function is accessible.
