Day 5 - Classes

You diden't think classes would be that simple, did you?

While writing your own class you may have ran into some annoyances, mainly setting all of those variables and changing them in a systematic manor

Functions

class Car { std::string color; std::string brand; int distanceTraveled; int numberOfWheels; int driveMiles(int miles) { distanceTraveled = distanceTraveled + miles return distanceTraveled; } } ...inside the main function... Car myCar; myCar.distanceTraveled = 10; std::cout << myCar.driveMiles(5) << std::endl; Output: 15

Ok so you probobly have a couple of questoins, mainly, what "driveMiles", "miles", and "return"

Lets disect this function: int signifies that the function returns a variable of type int.
miles is a variable local to the function, if there was another function inside the car class, it would NOT be able to access the value of "miles". In other words the variable is only accessable inside the {}. This variable contains the value we passed into the function, in this case 5.
return - this return a value back, in this case the new distanceTraveled. In this case, it prints out that value to the console but the value can also be saved to a variable.

Constructors

There is another important type of functoin that CONSTRUCTS the functoin

A constructor does not return anything and as such does not need a return type

The constructor also has to match the name of the class

The constructor is run once when you make the variable and typically setup all of the variables in the class

Private vs. Public

Private: The private class makes it so that the variables are inaccessable outside of the class. Why would you do this? This is done so that the person using your class (in real programming, other people tend to use classes you make without fully undertsanding everything) does not make a mistake.

Public:You can use the variables outside of the class. This is the default conditoin that we have been using

class Car { private: std::string color; std::string brand; int distanceTraveled; int numberOfWheels; public: Car(std::string c, std::string b, int d, int n) { color = c; brand = b; distanceTraveled = d; numberOfWheels = n; } int driveMiles(int miles) { distanceTraveled = distanceTraveled + miles; return distanceTraveled; } }; ...inside the main function... Car myCar("red", "Toyota", 0, 4); myCar.driveMiles(10); NOTE: trying to do myCar.color = "red"; WILL result in an error

Part B: In part B you will make a seonond constructor for the class, the second constructor has to take in different peramiters as this is how the code tells apart witch one to use when making a class. Example:

class Car { private: std::string color; std::string brand; int distanceTraveled; int numberOfWheels; public: Car(std::string c, std::string b, int d, int n) { //Constructor 1 color = c; brand = b; distanceTraveled = d; numberOfWheels = n; } Car(std::string c, std::string b) { // Constructor 2 color = c; brand = b; distanceTraveled = 0; numberOfWheels = 4; } Car() { // Constructor 3 color = "black"; brand = "Toyota"; distanceTraveled = 0; numberOfWheels = 4; } }; ...inside the main function... Cheatsheet Day 5

        Making a class:
        
        class Car {
          type variableName;
        }
        
        
        Accessing a class:
        Class className;
        className.variableName = value;

        Functions:
        - Syntax:
          returnType functionName(parameters) { ... }
        - Example:
          
          int driveMiles(int miles) {
            distanceTraveled += miles;
            return distanceTraveled;
          }
          
        - Key Points:
          - returnType: Specifies the type of value the function returns.
          - parameters: Variables passed into the function.
          - return: Sends a value back to the caller.

        Constructors:
        - Syntax:
          ClassName(parameters) { ... }
        - Example:
          
          Car(std::string c, std::string b, int d, int n) {
            color = c;
            brand = b;
            distanceTraveled = d;
            numberOfWheels = n;
          }
          
        - Key Points:
          - No return type.
          - Automatically called when an object is created.
          - Used to initialize class variables.

        Overloading:
        - Syntax:
          Multiple functions/constructors with the same name but different parameters.
        - Example:
          
          Car(std::string c, std::string b) {
            color = c;
            brand = b;
            distanceTraveled = 0;
            numberOfWheels = 4;
          }
          
        - Key Points:
          - Differentiated by the number and type of parameters.
          - Allows flexibility in how functions/constructors are called.
